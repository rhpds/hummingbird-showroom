== Welcome to Project Hummingbird

Welcome to the **Zero CVE Hummingbird Workshop**! This comprehensive two-module workshop is designed to help you master the skills needed to build, deploy, and manage minimal, hardened container environments using Red Hat's innovative Project Hummingbird initiative.

**Project Hummingbird** represents Red Hat's strategic approach to delivering "zero-CVE" container base images built from Fedora components. Launched in November 2025, Hummingbird addresses critical challenges in modern software supply chains by providing micro-sized, hardened runtime images with complete transparency through SBOMs (Software Bill of Materials).

== Why Hummingbird Matters

Traditional container base images, even minimal variants like UBI Minimal, include packages that aren't needed at runtime. These unnecessary components create:

* **Larger attack surfaces** with more CVEs to patch
* **Slower deployments** due to image size
* **Compliance challenges** for supply chain security
* **Higher infrastructure costs** from bloated images

**Hummingbird's Solution:**

* **Zero-CVE at Ship Time**: Images delivered free of known vulnerabilities
* **Micro-Sized**: JRE runtime ~185MB, Node.js < 100MB, Go < 25MB (vs ~400MB+ for full UBI)
* **Enterprise Grade**: Built on Fedora/RHEL foundation with Red Hat support
* **Complete Transparency**: Every image ships with SBOMs for compliance
* **Production Ready**: Runtime images for .NET, Go, Java, Node.js, Python, databases, and web servers

== What You'll Learn

=== Module 1: Container Development Environment with Hummingbird

In this hands-on module, you'll build and secure container images using a pre-configured developer environment:

* **Multi-Stage Builds**: Build production images using UBI builder + Hummingbird runtime
* **Security Tools**: Use cosign, syft, and grype for image signing and scanning
* **Security Workflow**: Scan for CVEs, generate SBOMs, sign and verify images
* **Size Optimization**: Compare image sizes and vulnerability counts
* **SELinux Hardening**: Generate custom SELinux policies with udica for Hummingbird containers
* **Advanced SELinux**: Device access policies, iteration workflows, and policy distribution patterns
* **Optional -- chunkah**: Content-based layer splitting for optimised image pulls
* **Optional -- Multi-Language Builds**: Production Containerfiles for Node.js, Quarkus Native, Python, and Go
* **Optional -- The Aha Moment**: Demonstrating chunkah value with CVE simulations and layer digest diffs
* **Optional -- Tomcat on Hummingbird**: Multi-stage Tomcat builds with RPM and upstream approaches, production hardening, and chunkah layer tagging

.*Expected Outcomes*
By the end of Module 1, you'll be able to build, scan, sign, and harden Hummingbird-based container images locally with production-ready SELinux policies.

=== Module 2: Building Hummingbird Images with Shipwright on OpenShift

This module focuses on platform-level image building and deployment:

* **Cloud Native Buildpacks**: Implement source-to-image builds without Dockerfiles
* **Hummingbird Integration**: Configure Buildpacks to use Hummingbird runtime images
* **Custom Strategies**: Create multi-language BuildStrategy patterns for development teams
* **Security Pipeline**: Integrate SBOM generation, scanning, and signing into builds
* **Production Deployment**: Deploy built images to OpenShift namespaces
* **SELinux CI/CD**: Automate udica policy generation with Tekton and distribute via MachineConfig
* **Optional -- chunkah Pipeline**: Automated content-based layer splitting in CI/CD
* **Optional -- RHTAS**: Keyless image signing and SBOM attestation with Red Hat Trusted Artifact Signer
* **Optional -- The Immutable Fortress**: Zero-CVE enforcement with ACS admission control and Hummingbird distroless images

.*Expected Outcomes*
By the end of Module 2, you'll have working Shipwright builds deploying Hummingbird-based applications with automated security controls and SELinux policies.
