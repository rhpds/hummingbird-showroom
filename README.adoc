= Zero CVE Hummingbird Workshop

== Introduction

This repository hosts the lab guide for the **Zero CVE Hummingbird Workshop**, built with https://antora.org[Antora] and designed for the https://github.com/rhpds/showroom_template_nookbag[Red Hat Showroom] lab platform on the Red Hat Demo Platform (RHDP).

**Project Hummingbird** represents Red Hat's strategic approach to delivering "zero-CVE" container base images built from Fedora components. This workshop provides hands-on experience building, securing, and deploying minimal, hardened container environments using Hummingbird runtimes, SELinux policy generation with udica, and platform-level builds with Shipwright on OpenShift.

== Who Will Benefit Most?

**Developers (Intermediate to Advanced)** -- Those responsible for building and securing container images locally using Podman, Buildah, and Skopeo. These users will learn multi-stage builds with Hummingbird runtimes, security scanning, SBOM generation, image signing, and custom SELinux policy generation with udica.

**Platform Engineers / DevOps (Intermediate to Advanced)** -- Those responsible for platform-level container build and deployment infrastructure on OpenShift. These users will learn Shipwright Builds, Cloud Native Buildpacks, custom BuildStrategies, security pipelines, and automated SELinux policy CI/CD with Tekton.

== Workshop Content

*Duration:* 90 minutes (45 minutes per module) + optional exercises

=== Module 1: Developer Environment (~45 min)

* **1.1 Building with Hummingbird** -- Multi-stage builds with UBI builder + Hummingbird runtime
* **1.2 Security Scanning & SBOMs** -- cosign, syft, grype for image signing and scanning
* **1.3 SELinux Fundamentals with udica** -- Custom SELinux policy generation from `podman inspect`
* **1.4 Advanced SELinux Hardening** -- Device access, `--append-rules`, iteration workflows, policy distribution

=== Module 2: Shipwright and Buildpacks (~45 min)

* **2.1 Cloud Native Buildpacks** -- Source-to-image builds without Dockerfiles
* **2.2 Custom Multi-Language Strategies** -- Auto-detection for Node.js, Python, Java, Go
* **2.3 Security Pipeline & Production** -- SBOM generation, vulnerability scanning, image signing
* **2.4 SELinux Policy CI/CD with Tekton** -- Automated udica policy generation in pipelines
* **2.5 The Immutable Fortress -- Zero-CVE with ACS Enforcement** -- ACS admission control, policy enforcement, vulnerability scanning

=== Optional Exercises

* **1.5 Content-Based Layer Splitting (chunkah)** -- Optimise image layers for pull deduplication
* **1.6 Multi-Language Builds with Hummingbird**
* **1.7 The Aha Moment -- Demonstrating chunkah Value**
* **1.8 Layering Tomcat on Hummingbird**
* **2.5 chunkah in CI/CD Pipelines** -- Automated layer splitting in Tekton pipelines
* **2.6 Keyless Signing with RHTAS**
* **2.7 Automated Dependency Updates with Renovate + Podman** -- Renovate Bot as Tekton pipeline with Buildah/Skopeo toolchain

=== Appendix: Environment Setup

* **Appendix A** -- RHEL Developer Environment (Podman, Buildah, Skopeo, Podman Desktop)
* **Appendix B** -- OpenShift Platform Environment (OpenShift Pipelines, Builds for OpenShift)

Both modules function independently. Users can take Module 1 only, Module 2 only, or both.

== Local Development

=== Prerequisites

* `podman` or `docker` (for building the Antora site locally)
* `git`

=== Clone the Repository

[source,bash]
----
git clone https://github.com/rhpds/zero-cve-hummingbird-showroom.git
cd zero-cve-hummingbird-showroom
----

=== Preview Locally

[source,bash]
----
podman run --rm --name antora -v $PWD:/antora -p 8080:8080 -i -t ghcr.io/juliaaano/antora-viewer
----

On SELinux systems, append `:z` to the volume mount.
